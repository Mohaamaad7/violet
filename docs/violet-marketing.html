الأسئلة الشائعة

كل ما تريد معرفته

ما الفرق بين Violet ومنصات التجارة الإلكترونية الأخرى؟ ▾
Violet مبني خصيصاً للسوق العربي والمصري على وجه الخصوص. يدعم جميع بوابات الدفع المصرية (Kashier, Paymob, Vodafone Cash, InstaPay, Fawry)، ومبني بـ RTL أولاً، مع نظام مؤثرين متكامل نادر في المنصات الجاهزة. الكود مفتوح تماماً لأي تعديل مستقبلي بدون قيود ترخيص.
هل يحتاج النظام إلى خادم خاص؟ ▾
لا — يعمل على أي Shared Hosting يدعم PHP 8.2+ وMySQL 8. للأداء الأمثل نوصي بـ VPS يدعم Redis وNginx. النظام يأتي مع إعدادات Apache و.htaccess جاهزة، وملف nginx.conf.example للمواقع الإنتاجية.
هل يدعم النظام اللغة الإنجليزية أيضاً؟ ▾
نعم — النظام ثنائي اللغة بالكامل (عربي/إنجليزي) مع تبديل فوري من الواجهة. نظام الترجمة DB-Backed يتيح للمسؤول تعديل أي نص من لوحة الإدارة بدون لمس الكود، مع كاش ذكي وكشف تلقائي للغة المفضلة.
كم يستغرق تثبيت النظام وإطلاقه؟ ▾
مع الإعداد الصحيح يمكن إطلاق المتجر في ساعات. قائمة النشر تشمل: git pull، composer install، php artisan migrate، npm run build. البيانات الجغرافية (27 محافظة و207 مدينة) تُحمّل بـ Seeder واحد. بوابات الدفع تُعدّ من لوحة الإدارة مباشرة.
ما طبيعة نظام المؤثرين؟ هل هو مدمج كاملاً؟ ▾
نعم — نظام مؤثرين متكامل من الألف للياء: تقديم طلب، مراجعة وقبول/رفض، إنشاء أكواد خصم مخصصة، تسجيل عمولات تلقائي عند كل طلب مدفوع، طلبات صرف الأرباح (بنك/Vodafone Cash/InstaPay)، ولوحة تحكم مستقلة للمؤثر يرى فيها كل إحصائياته.
هل يمكن التخصيص والإضافة عليه؟ ▾
بالتأكيد — Laravel مشهور بمرونته. النظام مبني بمعمارية Service Layer فاصلة بين منطق الأعمال والعرض. يمكن إضافة بوابات دفع جديدة عبر PaymentGatewayInterface، إضافة Filament Resources بسطور قليلة، وتوسيع نظام الترجمة لأي لغة.
